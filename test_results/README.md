# Test Results

This directory contains output files from test runs and validation.

---

## Files

### Test Output (JSON)

1. **[fix.json](fix.json)** - Fix validation results
   - Validates all recent fixes
   - Checks consistency across components
   - Generated by patch validation tests

2. **[integrated_scan_results.json](integrated_scan_results.json)** - Integration test results
   - Full pipeline test output
   - Catastrophe detection results
   - Attack + scan integration data

3. **[monodromy_test_results.json](monodromy_test_results.json)** - Monodromy test output
   - Monodromy detection results
   - Validation of existence checks
   - Multi-slab integration data

---

## File Formats

### JSON Structure

All JSON files follow a similar structure:

```json
{
  "test_name": "Test Description",
  "timestamp": "2025-01-23T...",
  "status": "PASSED",
  "results": {
    // Test-specific data
  }
}
```

---

## Usage

### Viewing Results

```bash
# Pretty print JSON
cat test_results/fix.json | python -m json.tool

# Search for specific fields
jq '.results' test_results/integrated_scan_results.json

# Check test status
grep -h "status" test_results/*.json
```

### Comparing Results

```bash
# Compare two test runs
diff test_results/old_results.json test_results/new_results.json

# Check for regressions
python scripts/compare_test_results.py
```

## Maintenance

### Cleanup

```bash
# Remove old results (keep recent)
find test_results/ -name "*.json" -mtime +30 -delete

# Archive old results
tar -czf test_results_archive_$(date +%Y%m%d).tar.gz test_results/
```

### Regenerating Results

```bash
# Run tests to regenerate
pixi run pytest tests/ --json-report --json-report-file=test_results/output.json

# Or run specific tests
python tests/test_monodromy_existence.py > test_results/monodromy_test_results.json
```

- ✅ **Version Control**: Consider `.gitignore` for large result files
